# Nanobananaへの生成プロンプト

あなたはプロのサイエンティフィック・イラストレーター兼プレゼンテーションデザイナーです。
バイオインフォマティクス初心者向け講義スライドに使用する、一連の解説図を作成してください。

## 全体デザイン指針
- **スタイル**: フラットデザイン、ミニマル、清潔感のあるベクターアート風。あとで編集や加工がしやすいよう、背景は白または単色にしてください。
- **配色**: 統一されたカラーパレットを使用してください（例：信頼感のある青、ティール、グレーを基調とし、強調色にオレンジやアクセントカラーを使用）。
- **文字**: 図中に文字を入れる場合は、必ず「日本語」で記述してください。
- **ターゲット**: 初心者が直感的に理解できる、わかりやすさを最優先してください。

## 作成リクエストリスト

### 1. Paired-endシーケンスとマージの仕組み
DNA断片の両端から読み、真ん中でつなげるイメージ図。
- **描写内容**:
    1. 1本のDNA断片（Fragment）がある。
    2. 左端から読む「Forward Read（R1）」と、右端から読む「Reverse Read（R2）」の矢印。
    3. R1とR2が中央で重なっている（Overlap）。
    4. 重なりを利用して、1本の長い配列として結合（Merge/Stitching）される様子。
- **ラベル（日本語）**: 「DNA断片」「Forward Read」「Reverse Read」「オーバーラップ」「マージ（結合）」

### 2. DADA2による品質管理プロセス
ノイズ除去の工程を示すフロー図。
- **描写内容**: 左から右へ流れるベルトコンベアのような工程。
    1. 汚れたデータ（低クオリティ）が入る。
    2. **フィルタリング**: クオリティの低いものが捨てられる。
    3. **トリミング**: 両端のプライマー配列が切り落とされる。
    4. **デノイズ**: エラーが修正される（ピカピカになるイメージ）。
    5. **マージ**: 2本が結合する。
    6. **キメラ除去**: おかしなつぎはぎ配列が除外される。
- **ラベル（日本語）**: 「入力データ」「フィルタリング」「トリミング」「デノイズ（エラー補正）」「マージ」「キメラ除去」「高品質な配列（ASV）」

### 3. ASVとOTUの違い
従来の「似ているものをまとめる」手法と、最新の「違いを見分ける」手法の対比。
- **描写内容**:
    - **OTU（従来）**: 97%似ている配列たち（赤、少し薄い赤、濃い赤など微妙に違う丸）を、ひとまとめの「赤」として扱う（大雑把）。
    - **ASV（DADA2）**: 1塩基の違い（赤と、中心に黒点がある赤など）を、統計モデルを使って「別物」として正確に見分ける（高解像度）。
- **ラベル（日本語）**:
    - 左側：「OTU (97%類似でまとめる = 解像度が低い)」
    - 右側：「ASV (1塩基の違いを区別 = 高解像度)」

### 4. 系統分類（Taxonomy）のイメージ
未知の配列をデータベースと照合して名前を当てる図。
- **描写内容**:
    1. 左側に「自分の配列（Query）」を持つ研究者キャラクター。
    2. 中央に巨大な「辞書（データベース：Silvaなど）」の本棚。
    3. 右側に、照合の結果判明した「菌名（例：E. coli）」が表示される。
- **ラベル（日本語）**: 「自分の配列」「データベース照合」「菌種の同定」

### 5. Alpha多様性とBeta多様性
多様性解析の違いをコップとビー玉（または細菌キャラクター）で例える図。
- **描写内容**:
    - **Alpha多様性**: 1つのコップの中に、カラフルな種類の違う菌がたくさん入っている様子にフォーカス。「1つの環境内の種類の多さ」。
    - **Beta多様性**: コップAとコップBを並べ、中身の色合いが「似ているか、似ていないか」を比較している様子（距離を測る）。
- **ラベル（日本語）**:
    - Alpha多様性：「1サンプル内の多様性（種類の多さ）」
    - Beta多様性：「サンプル間の違い（構成の類似度）」
